# Support Matrix

This app is a static, browser-only validation workbench. JSON-like formats use in-app validators, and XML/XSD validation uses a bundled libxml2/xmllint WebAssembly engine so complex XSD 1.0 schemas can be checked without a backend.

All schema formats can be inspected in the Schema Summary tab. The summary is a best-effort hierarchy view for human scanning; validation diagnostics remain the source of truth for pass/fail behavior.

## Product Workflow

- Editors start empty, with no bundled fixture selected in the product UI.
- Validation reruns automatically after schema or message edits when Auto validate is enabled, and late results from older edits are ignored.
- Schema format detection can switch the schema selector for high-confidence signatures such as XSD, GraphQL SDL, Protobuf, OpenAPI, Avro, table schemas, and key-value rules.
- Manual schema format selection is preserved until the schema text changes again.
- The Schema Summary tab starts with required and optional fields visible; ordering, data types, limits, descriptions, and warnings can be toggled on.
- For XSD, the Sources tab accepts additional `.xsd` files for `xs:include` and `xs:import`; validation reruns when any source changes.

## XSD Note

The XSD adapter runs the XML instance and in-memory schema bundle through libxml2/xmllint compiled to WebAssembly. `xs:include` and `xs:import` work when the related schemas are added in the Sources tab; unresolved references, namespace mismatches, and schema compile failures fail closed with diagnostics. The custom TypeScript XSD parser is still used for the Schema Summary tab and as a fallback if the WebAssembly engine cannot load. The static browser app does not automatically fetch local file paths or remote URLs from `schemaLocation`, so related schemas must be supplied by the user.
