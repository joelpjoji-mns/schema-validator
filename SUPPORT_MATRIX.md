# Support Matrix

This app is a static, browser-only validation workbench. It is strongest for JSON-like schema validation and intentionally lighter for XSD because full W3C XSD validation requires a dedicated XSD engine, usually WASM or a backend service.

All schema formats can be inspected in the Schema Summary tab. The summary is a best-effort hierarchy view for human scanning; validation diagnostics remain the source of truth for pass/fail behavior.

## Product Workflow

- Editors start empty, with no bundled fixture selected in the product UI.
- Validation reruns automatically after schema or message edits when Auto validate is enabled, and late results from older edits are ignored.
- Schema format detection can switch the schema selector for high-confidence signatures such as XSD, GraphQL SDL, Protobuf, OpenAPI, Avro, table schemas, and key-value rules.
- Manual schema format selection is preserved until the schema text changes again.
- The Schema Summary tab can toggle ordering, required fields, optional fields, data types, limits, descriptions, and warnings.
- For XSD, the Sources tab accepts additional `.xsd` files for `xs:include` and `xs:import`; validation reruns when any source changes.

## XSD Note

The XSD adapter is deliberately a practical XSD-lite validator, not a full standards processor. It catches common complex XML message problems and highlights both XML and schema locations. `xs:include` and `xs:import` work when the related schemas are added in the Sources tab; unresolved references, namespace mismatches, and duplicate declarations fail closed with diagnostics. `xs:simpleContent` extension is validated as a simple text value plus extension attributes, while broader derivation such as `xs:complexContent` and `xs:simpleContent` restriction still fails closed. The static browser app does not automatically fetch local file paths or remote URLs from `schemaLocation`. Full XSD support would need a schema-aware WASM validator or an optional backend service while keeping this static GitHub Pages app as the UI.
